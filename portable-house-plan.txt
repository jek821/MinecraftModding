WHAT IS NBT?
------------
NBT (Named Binary Tag) is Minecraft's native format for saving data to disk,
to items, and over the network. In Java it appears as CompoundTag — a typed
HashMap. You write tag.putInt("x", 10) and read back tag.getInt("x"). Every
piece of persistent data in the game uses it: chunk blocks, entity health, chest
inventories, sign text. When you call StructureTemplate.save() or
BlockEntity.saveToItem(), they produce a CompoundTag you can store anywhere.


INTENDED BEHAVIOUR (FULL LOOP)
-------------------------------
1.  Player places the portable house block inside or near their house.
2.  Player right-clicks the block → a selection GUI opens.
3.  In the GUI the player sets min corner offsets and size (relative to the block)
    and optionally adds block IDs to filter (e.g. "minecraft:dirt").
4.  Player clicks Capture in the GUI.
    - The region is snapshotted into the block entity.
    - Blocks in the region (except filtered ones) are removed from the world.
5.  Player breaks the block to pick it up as an item carrying all the data.
6.  Player goes somewhere new and places the item.
    - The terrain in that space is saved first (terrainData).
    - The house is rebuilt around the new position.
      Filtered blocks are skipped on restore (treated as transparent).
    - The portable house block stays placed inside the restored house.
7.  Player breaks the block again to pick it up.
    - The house blocks disappear.
    - The terrain saved in step 6 is restored.
8.  Repeat from step 6.

KEY CONCEPT — CompoundTag
  Think of CompoundTag as a HashMap<String, Object> that Minecraft knows how to
  write to disk. tag.putInt("key", value), tag.putString("key", value), etc.
  Read back with tag.getInt("key"). Chests use it for inventory, signs use it
  for text, you'll use it for house structure data.

  public Vec3i relativeMin  = new Vec3i(-5, -3, -5);   // offset from block to min corner

Save and load all of them in saveAdditional / loadAdditional:

  // saveAdditional:
  super.saveAdditional(tag, registries);
  tag.putInt("minX", relativeMin.getX());
  tag.putInt("minY", relativeMin.getY());
  tag.putInt("minZ", relativeMin.getZ());
  tag.putInt("sizeX", size.getX());
  tag.putInt("sizeY", size.getY());
  tag.putInt("sizeZ", size.getZ());
  ListTag filterTag = new ListTag();
  blockFilter.forEach(id -> filterTag.add(StringTag.valueOf(id)));
  tag.put("blockFilter", filterTag);
  if (houseData   != null) tag.put("houseData",   houseData);
  if (terrainData != null) tag.put("terrainData", terrainData);

  // loadAdditional:
  super.loadAdditional(tag, registries);
  relativeMin = new Vec3i(tag.getInt("minX"), tag.getInt("minY"), tag.getInt("minZ"));
  size        = new Vec3i(tag.getInt("sizeX"), tag.getInt("sizeY"), tag.getInt("sizeZ"));
  blockFilter = new ArrayList<>();
  tag.getList("blockFilter", Tag.TAG_STRING)
     .forEach(t -> blockFilter.add(t.getAsString()));
  houseData   = tag.contains("houseData")   ? tag.getCompound("houseData")   : null;
  terrainData = tag.contains("terrainData") ? tag.getCompound("terrainData") : null;

KEY CONCEPT — Relative vs Absolute positions
  Store the region as an offset relative to the portable house block, not as
  absolute world coordinates. When you place the item somewhere new, compute:
    BlockPos absoluteMin = newPlacedPos.offset(relativeMin);
  If you stored absolute coordinates, placing the house elsewhere would rebuild
  it back at the original world position instead of the new one.

  PayloadTypeRegistry.playS2C().register(

      UpdatePortableHouseRegionPayload.TYPE, UpdatePortableHouseRegionPayload.CODEC);

  // Handle the incoming C2S packet on the server
  ServerPlayNetworking.registerGlobalReceiver(
      UpdatePortableHouseRegionPayload.TYPE, (payload, context) -> {
          context.server().execute(() -> {
              ServerLevel level = context.player().serverLevel();
              if (!(level.getBlockEntity(payload.pos())
                      instanceof PortableHouseBlockEntity be)) return;

              be.size         = payload.size();

          });
      });

STEP 7 — Capture the structure
Goal: Snapshot the house into the block entity and remove it from the world.
Files: ~ PortableHouseBlock.java
================================================================================

Add a static captureHouse() method to PortableHouseBlock (called from the packet
handler registered in Step 6c):

  public static void captureHouse(ServerLevel level, BlockPos pos,
                                   PortableHouseBlockEntity be) {
      BlockPos absoluteMin = pos.offset(be.relativeMin);
      BlockPos sizePos     = new BlockPos(be.size.getX(), be.size.getY(), be.size.getZ());

      // Snapshot the region
      StructureTemplate template = new StructureTemplate();
      template.fillFromWorld(level, absoluteMin, sizePos, false, Blocks.STRUCTURE_VOID);
      be.houseData = template.save(new CompoundTag());

      // Remove captured blocks from the world (the house now lives in the item)
      // Skip filtered blocks — they stay in the world as part of the natural terrain
      Set<Block> filtered = be.blockFilter.stream()
          .map(id -> BuiltInRegistries.BLOCK.get(Identifier.parse(id)))
          .filter(b -> b != Blocks.AIR)
          .collect(Collectors.toSet());

      BlockPos.betweenClosed(absoluteMin,
              absoluteMin.offset(be.size.getX()-1, be.size.getY()-1, be.size.getZ()-1))
          .forEach(p -> {
              if (p.equals(pos)) return; // never remove the portable house block itself
              BlockState bs = level.getBlockState(p);
              if (!filtered.contains(bs.getBlock())) {
                  level.removeBlock(p, false);
              }
          });

      be.setChanged();
      level.sendBlockUpdated(pos, level.getBlockState(pos), level.getBlockState(pos), 3);
      // Notify the player however you like — chat message, sound, etc.
  }

KEY CONCEPT — Block filtering on restore
  The houseData NBT contains ALL blocks in the region including filtered ones.
  The filtering is applied on RESTORE using a StructureProcessor (see Step 10).
  The processor simply returns null for any block whose ID is in the filter list,
  which causes placeInWorld() to skip it (leave that position as-is).

  StructureProcessor subclass:
    class BlockFilterProcessor extends StructureProcessor {
        private final Set<Block> filtered;
        BlockFilterProcessor(Set<Block> f) { this.filtered = f; }

        @Override
        public StructureBlockInfo processBlock(LevelReader level,
                BlockPos offset, BlockPos pos,
                StructureBlockInfo original, StructureBlockInfo relative,
                StructurePlaceSettings settings) {
            return filtered.contains(relative.state().getBlock()) ? null : relative;
        }

        @Override
        protected StructureProcessorType<?> getType() {
            return StructureProcessorType.NOOP;
        }
    }

STEP 10 — Save terrain then restore house on place
Goal: When the item is placed, snapshot the existing terrain first, then rebuild
      the house. The terrain snapshot lets you restore it when the house leaves.
Files: ~ block/PortableHouseBlockItem.java
================================================================================

  @Override
  public InteractionResult place(BlockPlaceContext context) {
      InteractionResult result = super.place(context);
      if (!result.consumesAction()) return result;

      Level level = context.getLevel();
      if (level.isClientSide()) return result;

      BlockPos placedPos = context.getClickedPos();
      if (!(level.getBlockEntity(placedPos) instanceof PortableHouseBlockEntity be))
          return result;
      if (be.houseData == null) return result;

      BlockPos absoluteMin = placedPos.offset(be.relativeMin);
      BlockPos sizePos     = new BlockPos(be.size.getX(), be.size.getY(), be.size.getZ());

      // 1. Save what's currently here so we can restore it on pickup
      StructureTemplate terrain = new StructureTemplate();
      terrain.fillFromWorld(level, absoluteMin, sizePos, false, Blocks.STRUCTURE_VOID);
      be.terrainData = terrain.save(new CompoundTag());

      // 2. Restore the house, applying the block filter via a StructureProcessor
      Set<Block> filtered = be.blockFilter.stream()
          .map(id -> BuiltInRegistries.BLOCK.get(Identifier.parse(id)))
          .filter(b -> b != Blocks.AIR)
          .collect(Collectors.toSet());
      StructurePlaceSettings settings = new StructurePlaceSettings()
          .addProcessor(new BlockFilterProcessor(filtered));

      StructureTemplate house = new StructureTemplate();
      house.load(level.registryAccess(), be.houseData);
      house.placeInWorld(level, absoluteMin, absoluteMin, settings, level.getRandom(), 2);

      be.setChanged();
      level.sendBlockUpdated(placedPos, level.getBlockState(placedPos),
          level.getBlockState(placedPos), 3);

      return result;
  }

NOTE: absoluteMin uses placedPos (the new placement position), not the original
capture position. relativeMin is stored relative to the block, so
placedPos.offset(relativeMin) always gives the correct absolute min corner
regardless of where in the world the item is placed.

STEP 11 — Restore terrain on pickup
Goal: When the block is broken, remove the house and put back whatever was
      in that space before the house arrived.
Files: ~ PortableHouseBlock.java
================================================================================

Override onRemove() in PortableHouseBlock:

  @Override
  public void onRemove(BlockState state, Level level, BlockPos pos,
                       BlockState newState, boolean movedByPiston) {
      if (!level.isClientSide() && !state.is(newState.getBlock())) {
          if (level.getBlockEntity(pos) instanceof PortableHouseBlockEntity be) {
              BlockPos absoluteMin = pos.offset(be.relativeMin);

              if (be.terrainData != null) {
                  // Restore the terrain that was here before the house was placed
                  StructureTemplate terrain = new StructureTemplate();
                  terrain.load(level.registryAccess(), be.terrainData);
                  terrain.placeInWorld(level, absoluteMin, absoluteMin,
                      new StructurePlaceSettings(), level.getRandom(), 2);

              } else if (be.houseData != null) {
                  // First pickup after original capture: no terrain to restore,
                  // just clear the region so the house vanishes
                  BlockPos.betweenClosed(
                      absoluteMin,
                      absoluteMin.offset(be.size.getX()-1, be.size.getY()-1, be.size.getZ()-1)
                  ).forEach(p -> {
                      if (!p.equals(pos)) level.removeBlock(p, false);
                  });
              }
          }
      }
      super.onRemove(state, level, pos, newState, movedByPiston); // MUST be last
  }

WARNING: Always call super.onRemove() LAST. It destroys the block entity — if
you call it first, level.getBlockEntity(pos) returns null and you lose all data.
