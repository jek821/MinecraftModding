1
Create an empty BlockEntity class
Learn what a BlockEntity is and the minimum code it needs
▼
+ PortableHouseBlockEntity.java

Create PortableHouseBlockEntity.java in the block/ package. It needs to extend BlockEntity and override two methods for saving and loading data. Both methods can have empty bodies for now — the goal is just a class that compiles.

Methods to implement

  • Constructor (BlockEntityType<?> type, BlockPos pos, BlockState state) — passes all three up to super(...). You'll fill in the type argument in Step 3.
  • saveAdditional(CompoundTag tag, HolderLookup.Provider registries) — called automatically when Minecraft saves this block to disk. Leave empty for now.
  • loadAdditional(CompoundTag tag, HolderLookup.Provider registries) — called when Minecraft reads it back. Leave empty for now.
Key Concept — CompoundTag

Think of CompoundTag as a HashMap<String, Object> that Minecraft knows how to write to disk. You put things in with tag.putInt("key", value), tag.putString("key", value), etc., and read them back with tag.getInt("key"). This is how all block entity data is persisted — chests use it for inventory, signs use it for text, and you'll use it for house structure data.

Register the BlockEntityType
Learn how Minecraft's registry system works for block entities
▼
+ ModBlockEntities.java ~ JacobMod.java

Create ModBlockEntities.java modelled after ModBlocks.java. Register a BlockEntityType that links your new class to your block:

public static final BlockEntityType<PortableHouseBlockEntity> PORTABLE_HOUSE =
    Registry.register(
        BuiltInRegistries.BLOCK_ENTITY_TYPE,
        Identifier.fromNamespaceAndPath("jacobmod", "portable_house"),
        BlockEntityType.Builder
            .of(PortableHouseBlockEntity::new, ModBlocks.PORTABLEHOUSE)
            .build(null)
    );

Add a registerModBlockEntities() method (empty body is fine — calling it forces the class to load and run the static field above). Then call it from JacobMod.onInitialize().

Key Concept — BlockEntityType

The BlockEntityType is a registry entry that acts as a factory and validator. It says: "positions that hold a portable_house block are allowed to have a PortableHouseBlockEntity attached, and here's how to construct one." Minecraft uses it internally to create and look up block entities when loading chunks.

4
Add right-click interaction
Learn the server/client split and how to intercept player actions
▼
~ PortableHouseBlock.java

Override useWithoutItem() in PortableHouseBlock.java. For now just send a chat message to confirm it works:

@Override
protected InteractionResult useWithoutItem(
    BlockState state, Level level, BlockPos pos,
    Player player, BlockHitResult hit
) {
    if (level.isClientSide()) return InteractionResult.SUCCESS;

    player.sendSystemMessage(Component.literal("Right clicked portable house!"));
    return InteractionResult.SUCCESS;
}
Key Concept — Server vs Client

In Minecraft, the world runs on both the server (holds real game state) and the client (your screen, visual copy only). This method is called on both sides. The if (level.isClientSide()) return guard ensures your real logic only runs on the server, where it actually matters. Almost all block and item logic follows this pattern.

Test

Right-click the placed block without holding an item. You should see the message appear in chat.

5
Define the capture region
Learn BlockPos math and how to describe a 3D area
▼
~ PortableHouseBlockEntity.java ~ PortableHouseBlock.java

Add two BlockPos fields to PortableHouseBlockEntity: minCorner and maxCorner. These define the bounding box of the house. Start with a hardcoded radius:

// in PortableHouseBlock.useWithoutItem(), after the isClientSide guard:
PortableHouseBlockEntity be =
    (PortableHouseBlockEntity) level.getBlockEntity(pos);
if (be == null) return InteractionResult.FAIL;

be.minCorner = pos.offset(-10, -5, -10);
be.maxCorner = pos.offset(10, 10, 10);
be.setChanged(); // marks data as dirty — tells Minecraft to save it

Then in saveAdditional / loadAdditional, persist the two corners. Use tag.putInt("minX", minCorner.getX()) (and Y, Z) to save, and new BlockPos(tag.getInt("minX"), ...) to load. Don't forget to call super.saveAdditional() and super.loadAdditional() too.

Key Concept — setChanged()

Calling setChanged() on a block entity tells Minecraft "this data is dirty and needs to be written to disk on the next save." If you forget it, your data will be lost when the chunk unloads. Call it every time you modify a field on a block entity.

Test

Right-click the block. No crash = the region is being set and saved correctly. You can't see it yet, but the data is there.

6
Capture the structure with StructureTemplate
Use Minecraft's built-in system to snapshot the house
▼
~ PortableHouseBlock.java ~ PortableHouseBlockEntity.java

Add a CompoundTag houseData field to PortableHouseBlockEntity. Save/load it with tag.put("houseData", houseData) and houseData = tag.getCompound("houseData").

Then in the right-click handler, after setting the corners, scan the world into a StructureTemplate:

StructureTemplate template = new StructureTemplate();
BlockPos size = be.maxCorner.subtract(be.minCorner);
template.fillFromWorld(level, be.minCorner, size, false, Blocks.STRUCTURE_VOID);

be.houseData = template.save(new CompoundTag());
be.setChanged();
level.sendBlockUpdated(pos, state, state, 3); // syncs to client

player.sendSystemMessage(Component.literal("House captured!"));
Key Concept — StructureTemplate

StructureTemplate is the same system Minecraft uses for structure blocks and jigsaw structures. fillFromWorld() walks every block in the region and records its BlockState and its block entity NBT (chests with items, signs with text, etc.). template.save() converts it to a CompoundTag you can store anywhere — in your block entity, in an item, on disk.

Test

Right-click the block. You should see "House captured!" in chat. Verify the data was saved by breaking and replacing the block — the capture process shouldn't crash.

7
Copy block entity data into the dropped item
Bridge the gap between the placed block and the carried item
▼
~ PortableHouseBlock.java

Override getDrops() in PortableHouseBlock to attach the block entity's NBT to the dropped item stack. Minecraft provides a helper that does this automatically:

@Override
public List<ItemStack> getDrops(
    BlockState state, LootParams.Builder builder
) {
    BlockEntity be = builder.getOptionalParameter(LootContextParams.BLOCK_ENTITY);
    ItemStack stack = new ItemStack(this);
    if (be != null) {
        be.saveToItem(stack, builder.create(LootContextParamSets.EMPTY).getLevel().registryAccess());
    }
    return List.of(stack);
}
Key Concept — saveToItem()

BlockEntity.saveToItem() calls your saveAdditional() internally and stores everything under a special BlockEntityTag component on the item stack. This is exactly how vanilla handles chests keeping their inventory when broken — you don't need to manually copy individual fields.

Test

Right-click to capture, then break the block. Hold the dropped item and run /data get entity @s SelectedItem in chat. You should see a large NBT blob containing houseData nested inside — that's your entire house structure sitting in the item.

9
Restore the structure on place
Read NBT from the item and reconstruct the house
▼
~ PortableHouseBlockItem.java

After super.place(context) succeeds, get the newly placed block's position from the context, then read the stored structure and replay it into the world:

InteractionResult result = super.place(context);
if (!result.consumesAction()) return result;

Level level = context.getLevel();
if (level.isClientSide()) return result;

BlockPos placedPos = context.getClickedPos();
PortableHouseBlockEntity be =
    (PortableHouseBlockEntity) level.getBlockEntity(placedPos);
if (be == null || be.houseData == null) return result;

StructureTemplate template = new StructureTemplate();
template.load(level.registryAccess(), be.houseData);

// minCorner was stored relative to the original placement —
// recalculate it relative to the new placement position
BlockPos offset = be.minCorner; // you'll refine this in Step 10
template.placeInWorld(
    level, offset, offset,
    new StructurePlaceSettings(),
    level.getRandom(), 2
);
return result;
Note — Relative Offsets

The min corner saved in Step 5 is an absolute world position. When the player places the item somewhere else, you need the corner relative to the portable house block, then add the new placement position. Update your block entity to store the offset as a relative Vec3i rather than an absolute position — e.g. (-10, -5, -10) — and reconstruct the absolute position at restore time using placedPos.offset(storedOffset).

Test

Right-click to capture the house, break the block, walk to an empty area, place the item. The house should rebuild around the new position.

10
Remove the original blocks on pickup
Make the house actually travel with the item
▼
~ PortableHouseBlock.java

Override onRemove() in PortableHouseBlock. This fires when the block leaves the world. Get the block entity, check that it has a captured structure, then clear every block in the saved region:

@Override
public void onRemove(
    BlockState state, Level level, BlockPos pos,
    BlockState newState, boolean movedByPiston
) {
    if (!level.isClientSide() && !state.is(newState.getBlock())) {
        PortableHouseBlockEntity be =
            (PortableHouseBlockEntity) level.getBlockEntity(pos);
        if (be != null && be.houseData != null) {
            BlockPos min = pos.offset(be.storedOffset);
            BlockPos max = min.offset(be.size); // size you recorded in step 5
            BlockPos.betweenClosed(min, max)
                .forEach(p -> level.removeBlock(p, false));
        }
    }
    super.onRemove(state, level, pos, newState, movedByPiston);
}
Important

Always call super.onRemove() after your logic. The super call detaches and destroys the block entity — if you call it first, level.getBlockEntity(pos) returns null and you lose the data.

Test

Build a small room, place and right-click the block to capture it, then break it. The room should disappear. Walk elsewhere and place the item — the room should reappear. This is the full loop working end-to-end.
