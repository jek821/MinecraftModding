Key Concept — NBT & CompoundTag

NBT (Named Binary Tag) is Minecraft's native format for saving data to disk, to items, and over the network. In Java it's the CompoundTag class — a typed HashMap. You write tag.putInt("x", 10) and read back tag.getInt("x"). Every piece of persistent data in the game — chunk blocks, chest inventories, sign text — is stored as NBT. When you override saveAdditional() and loadAdditional(), you're reading and writing NBT.

Key Concept — Relative vs Absolute positions

Store the region as an offset relative to the portable house block, not as absolute world coordinates. When the player places the item somewhere new, compute: absoluteMin = newPlacedPos.offset(relativeMin). If you stored absolute coordinates, placing the house elsewhere would rebuild it at the original world position instead of the new one.

PayloadTypeRegistry.playS2C().register(
    OpenPortableHouseGuiPayload.TYPE, OpenPortableHouseGuiPayload.CODEC);
PayloadTypeRegistry.playC2S().register(
    UpdatePortableHouseRegionPayload.TYPE, UpdatePortableHouseRegionPayload.CODEC);

// Handle the incoming C2S packet on the server
ServerPlayNetworking.registerGlobalReceiver(
    UpdatePortableHouseRegionPayload.TYPE, (payload, context) -> {
        context.server().execute(() -> {
            ServerLevel level = context.player().serverLevel();
            if (!(level.getBlockEntity(payload.pos())
                    instanceof PortableHouseBlockEntity be)) return;
            be.relativeMin = payload.relativeMin();
            be.size        = payload.size();
            be.blockFilter = new ArrayList<>(payload.blockFilter());
            be.setChanged();
            PortableHouseBlock.captureHouse(level, payload.pos(), be); // Step 7
        });
    });

        box.setValue(String.valueOf(initialValue));
        return addRenderableWidget(box);

~ PortableHouseBlock.java

Add a public static captureHouse() method to PortableHouseBlock. This is called from the packet handler registered in Step 6c.

public static void captureHouse(ServerLevel level, BlockPos pos,
                                  PortableHouseBlockEntity be) {
    BlockPos absoluteMin = pos.offset(be.relativeMin);
    BlockPos sizePos     = new BlockPos(be.size.getX(), be.size.getY(), be.size.getZ());

    // Snapshot the region
    StructureTemplate template = new StructureTemplate();
    template.fillFromWorld(level, absoluteMin, sizePos, false, Blocks.STRUCTURE_VOID);
    be.houseData = template.save(new CompoundTag());

    // Remove captured blocks from world — filtered blocks stay (they're background terrain)
    Set<Block> filtered = be.blockFilter.stream()
        .map(id -> BuiltInRegistries.BLOCK.get(Identifier.parse(id)))
        .filter(b -> b != Blocks.AIR)
        .collect(Collectors.toSet());

    BlockPos.betweenClosed(absoluteMin,
            absoluteMin.offset(be.size.getX()-1, be.size.getY()-1, be.size.getZ()-1))
        .forEach(p -> {
            if (p.equals(pos)) return; // never remove the portable house block itself
            if (!filtered.contains(level.getBlockState(p).getBlock()))
                level.removeBlock(p, false);
        });

    be.setChanged();
    level.sendBlockUpdated(pos, level.getBlockState(pos), level.getBlockState(pos), 3);
}
Key Concept — Block filtering via StructureProcessor

The houseData NBT stores all blocks in the region, including filtered ones. The filter is applied on restore using a StructureProcessor that returns null for any block in the filter list — placeInWorld() skips null results. This means filtered blocks are simply not placed at the destination; whatever is naturally at those positions shows through.

class BlockFilterProcessor extends StructureProcessor {
    private final Set<Block> filtered;
    BlockFilterProcessor(Set<Block> f) { this.filtered = f; }

    @Override
    public StructureBlockInfo processBlock(LevelReader level,
            BlockPos offset, BlockPos pos,
            StructureBlockInfo original, StructureBlockInfo relative,
            StructurePlaceSettings settings) {
        return filtered.contains(relative.state().getBlock()) ? null : relative;
    }

    @Override protected StructureProcessorType<?> getType() {
        return StructureProcessorType.NOOP;
    }
}

You'll use this class in Step 10 when calling placeInWorld().

Key Concept — StructureTemplate

StructureTemplate is the same system Minecraft uses for structure blocks. fillFromWorld() records every block's BlockState and block entity NBT (chest contents, sign text, etc.). template.save() turns it into a CompoundTag you can store anywhere.

@Override
public List<ItemStack> getDrops(BlockState state, LootParams.Builder builder) {
    BlockEntity be = builder.getOptionalParameter(LootContextParams.BLOCK_ENTITY);
    ItemStack stack = new ItemStack(this);
    if (be != null) {

    }
    return List.of(stack);
}
Key Concept — saveToItem()

Calls your saveAdditional() internally and stores everything under a special BlockEntityTag component on the item stack. This is exactly how vanilla chests keep their inventory when broken — no manual field copying needed. All five fields (relativeMin, size, blockFilter, houseData, terrainData) travel with the item automatically.

Save terrain then restore house on place
Snapshot what's there first, then rebuild the house
▼
~ PortableHouseBlockItem.java

Override place() in PortableHouseBlockItem. After the block is placed, save the terrain at that location and then restore the house structure:

@Override
public InteractionResult place(BlockPlaceContext context) {
    InteractionResult result = super.place(context);
    if (!result.consumesAction()) return result;

    Level level = context.getLevel();
    if (level.isClientSide()) return result;

    BlockPos placedPos = context.getClickedPos();
    if (!(level.getBlockEntity(placedPos) instanceof PortableHouseBlockEntity be)
            || be.houseData == null) return result;

    BlockPos absoluteMin = placedPos.offset(be.relativeMin);
    BlockPos sizePos     = new BlockPos(be.size.getX(), be.size.getY(), be.size.getZ());

    // 1. Save whatever is currently in this space
    StructureTemplate terrain = new StructureTemplate();
    terrain.fillFromWorld(level, absoluteMin, sizePos, false, Blocks.STRUCTURE_VOID);
    be.terrainData = terrain.save(new CompoundTag());

    // 2. Restore the house, skipping filtered blocks via StructureProcessor
    Set<Block> filtered = be.blockFilter.stream()
        .map(id -> BuiltInRegistries.BLOCK.get(Identifier.parse(id)))
        .filter(b -> b != Blocks.AIR)
        .collect(Collectors.toSet());
    StructurePlaceSettings settings = new StructurePlaceSettings()
        .addProcessor(new BlockFilterProcessor(filtered));

    StructureTemplate house = new StructureTemplate();
    house.load(level.registryAccess(), be.houseData);
    house.placeInWorld(level, absoluteMin, absoluteMin, settings, level.getRandom(), 2);

    be.setChanged();
    level.sendBlockUpdated(placedPos, level.getBlockState(placedPos),
        level.getBlockState(placedPos), 3);
    return result;
}
Note — Relative offsets

absoluteMin is computed from placedPos (the new placement position), not the original capture position. Because relativeMin is stored relative to the block, placedPos.offset(relativeMin) always gives the correct absolute min corner wherever in the world the item is placed.

~ PortableHouseBlock.java

Override onRemove() in PortableHouseBlock. This fires whenever the block is removed from the world for any reason.

@Override
public void onRemove(BlockState state, Level level, BlockPos pos,
                     BlockState newState, boolean movedByPiston) {
    if (!level.isClientSide() && !state.is(newState.getBlock())) {
        if (level.getBlockEntity(pos) instanceof PortableHouseBlockEntity be) {
            BlockPos absoluteMin = pos.offset(be.relativeMin);

            if (be.terrainData != null) {
                // Restore the terrain that was here before the house was placed
                StructureTemplate terrain = new StructureTemplate();
                terrain.load(level.registryAccess(), be.terrainData);
                terrain.placeInWorld(level, absoluteMin, absoluteMin,
                    new StructurePlaceSettings(), level.getRandom(), 2);

            } else if (be.houseData != null) {
                // First pickup after original capture — no terrain to restore,
                // just clear the region so the house vanishes
                BlockPos.betweenClosed(
                    absoluteMin,
                    absoluteMin.offset(be.size.getX()-1, be.size.getY()-1, be.size.getZ()-1)
                ).forEach(p -> {
                    if (!p.equals(pos)) level.removeBlock(p, false);
                });
            }
        }
    }
    super.onRemove(state, level, pos, newState, movedByPiston); // MUST be last
}
Critical — Call super last

Always call super.onRemove() after your logic. The super call destroys the block entity — if you call it first, level.getBlockEntity(pos) returns null and you lose all your data.
